The legacy platform: template syntax for cover is being removed. Please migrate garage_door to the modern template syntax.
Step 1 - Remove legacy configuration

Remove the garage_door template definition from the configuration.yaml cover: section.

Note: If you are using cover: !include <filename>.yaml in configuration.yaml, remove the cover definition from the included <filename>.yaml.
Step 2 - Add the modern configuration

Add new template definition inside configuration.yaml:

template:
- cover:
  - unique_id: garage_door_safe
    device_class: garage
    open_cover:
    - action: script.garage_open
    close_cover:
    - action: script.garage_close
    stop_cover:
    - action: script.garage_stop
    default_entity_id: cover.garage_door
    icon: "{% if is_state('input_boolean.garage_door_virtual_open', 'on') %}\n  mdi:garage-open\n{%
      else %}\n  mdi:garage\n{% endif %}"
    availability: '{{ is_state(''binary_sensor.garage_shelly_online'', ''on'') }}'
    name: Garage Door
    state: "{% if is_state('input_boolean.garage_door_virtual_open', 'on') %}\n  open\n{%
      else %}\n  closed\n{% endif %}"

Note: If there are any existing template: sections in your configuration, make sure to omit the template: line from the yaml above. There can only be 1 template: section in configuration.yaml. Also, ensure the indentation is aligned with the existing entities within the template: section.
Step 3 - Restart Home Assistant or reload template entities
